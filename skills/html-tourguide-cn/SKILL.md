# HTML Tour Guide CN

把旅行路线转换为一个极简、移动端优先、可离线打开的单文件 HTML 导览页。

## 何时使用

- 用户要把行程、City Walk、主题路线或一日游整理成 HTML 页面
- 用户希望页面适合手机打开、便于分享、尽量离线可用
- 用户希望每个地点都有可折叠详情和一个“导航”按钮

## 工作原则

- 默认极简，只展示用户当下最需要的信息
- 视觉风格安静、克制、像 Apple Notes / Apple Maps 的卡片界面
- 文案以“观察”和“线索”为主，不写长篇攻略
- 尽量不要追问，除非缺失信息会导致页面无法生成
- 生成结果必须是单文件 HTML，CSS 和 JavaScript 都写在同一个文件里

## 推荐输入

优先把信息整理成下面这种结构：

```yaml
title: ""
city: ""
duration: ""
transportation: ""
route_theme: ""
navigation_profile: "auto"
stops:
  - name: ""
    map_keyword: ""
    summary: ""
    context: ""
    observe:
      - ""
      - ""
    lat: 0
    lon: 0
    coord_type: "gcj02"
    nav_mode: "walk"
```

如果字段缺失，可以根据上下文合理补全。

## 页面结构

最终页面默认只保留两个主要部分：

1. 总览
2. 路线列表

不要默认添加这些栏目：

- 延伸阅读
- 摄影建议
- 美食推荐
- 打包清单
- 参考资料

只有用户明确要求时再加入。

## 总览部分

总览要短，且一眼能看懂：

- 导览标题
- 一句话主题
- 城市
- 时长
- 交通方式
- 路线顺序

示例风格：

```text
《浙江村》北京南城一日游
一条关于流动人口、服装市场与城市更新的路线。
城市：北京
时长：1 天
交通：地铁 + 步行
路线：大红门 → 海户屯 → 东罗园 → 石榴庄 → 大栅栏 → 天桥
```

## 路线列表

每个目的地都是一行可折叠卡片。

### 默认收起

只显示：

- 目的地名称
- 导航按钮

### 展开后

只显示：

- 一句话说明
- 简短背景
- 观察关键词

不要写成长篇文章。

### 交互规则

- 点击目的地行，展开或收起详情
- 点击“导航”按钮，只触发导航，不触发展开
- 默认同一时间只展开一个目的地
- 动效要轻量，不要复杂动画
- 默认不要展示多地图选择面板

## 导航策略

默认 `navigation_profile` 设为 `auto`。

- 有经纬度时，优先走路线规划
- 没有经纬度时，优先走关键词搜索
- 中国大陆路线优先使用 GCJ-02 坐标
- 坐标不确定时，宁可用关键词搜索，也不要乱猜位置

导航细节与 URL 模板见 [navigation reference](references/navigation.md)。

## 文案风格

保持简洁、克制、平静，偏观察型。

推荐用词：

- 路线
- 观察
- 街区
- 节点
- 线索
- 变化
- 迁移
- 更新
- 现场
- 痕迹

避免过度使用：

- 你必须
- 你一定要
- 非常重要
- 深入思考
- 充分体现
- 极具代表性
- 不容错过

## 视觉规范

建议使用浅色、卡片式布局，字体使用系统字体，避免依赖外部资源。

可参考这组变量：

```css
:root {
  --bg: #f5f5f7;
  --card: #ffffff;
  --text: #1d1d1f;
  --muted: #6e6e73;
  --line: rgba(0, 0, 0, 0.08);
  --accent: #007aff;
  --radius: 22px;
}
```

深色模式也可以支持，但不要影响默认清爽感。

## 生成前检查

- 页面只有“总览”和“路线列表”两个主要部分
- 每个目的地默认收起
- 收起状态只显示地点名和导航按钮
- 点击导航按钮不会触发展开或收起
- 页面是单文件 HTML
- 页面可离线打开
- 不依赖外部 CSS 库或外部 JavaScript 库
- 每个目的地都有地图关键词
- 如果能拿到坐标就补上
- 文字简短，不写成长篇攻略

## 输出

完成时，给出：

1. 生成的 HTML 文件
2. 一句简短说明，告诉用户如何打开和使用
